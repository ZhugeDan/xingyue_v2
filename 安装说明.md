运行环境要求： Ubuntu 22.04 LTS (2核2G以上) / Node.js 20.x / Python 3.10+ / Nginx / PM2。

0.在 2核 2G 的超小服务器上，Node.js 的打包过程非常吃内存。先建立虚拟内存。

```
# 1. 创建一个 2GB 的交换文件（充当临时内存）
sudo fallocate -l 2G /swapfile
# 2. 设置正确的安全权限
sudo chmod 600 /swapfile
# 3. 把这个文件格式化为 swap 格式
sudo mkswap /swapfile
# 4. 激活这个虚拟内存
sudo swapon /swapfile
# 5. 设置开机自动挂载
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

1.环境初始化： 在服务器安装 Nginx、Python3-venv、Node.js 及 PM2 进程守护工具。

```

# 安装 Node.js (这里使用 20.x 版本的官方脚本)
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs
# 安装 PM2
npm install pm2 -g
```

4.后端启动： 进入 backend/ 目录，创建并激活虚拟环境，执行 pip install -r requirements.txt。退回根目录，使用 pm2 start ecosystem.config.js 守护并启动 FastAPI 后端服务。

```
cd /opt/xingyue_v2/backend
# 创建纯净的 Python 虚拟环境（不使用 Conda，节省服务器内存）
python3 -m venv venv
# 激活环境并安装依赖
source venv/bin/activate
pip install -r requirements.txt
# 退回到项目根目录，使用PM2 脚本启动后端
cd /opt/xingyue_v2
nano ecosystem.config.js
# 这段配置完整粘贴
module.exports = {
  apps : [{
    name   : "diary-backend",
    cwd    : "./backend",
    script : "venv/bin/uvicorn",
    args   : "main:app --host 0.0.0.0 --port 8000",
    interpreter: "none"
  }]
}
###分割线，配置文件结束

pm2 start deploy/ecosystem.config.js
pm2 save
pm2 startup # 设置开机自启
```

5.反向代理与 SSL： 将项目内的 nginx.conf 软链接至 /etc/nginx/sites-enabled/，配置静态文件托管与 /api 代理。执行 certbot --nginx 自动配置 HTTPS 加密，最后通过 systemctl restart nginx 完成上线。

```
# 手写Nginx配置文件
nano nginx.conf
# 粘贴下面的配置文件
server {
    listen 80;
    # server_name _; # 如果有域名，把 _ 换成域名
    wangxingyue.cn; # 这里是我的域名。

    # 1. 托管前端打包后的纯静态文件
    location / {
        root /home/ubuntu/xingyue_v2/frontend/dist; #这里一定要小心，点击pwd，查看准确路径
        index index.html;
        try_files $uri $uri/ /index.html; # 解决刷新 404 问题
    }

    # 2. 反向代理后端 API
    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 3. 反向代理接口文档 (方便线上调试)
    location /docs {
        proxy_pass http://127.0.0.1:8000/docs;
    }
    location /openapi.json {
        proxy_pass http://127.0.0.1:8000/openapi.json;
    }
}
# 分隔符 此处脚本结束
# 替换系统Nginx默认配置并重启
sudo rm -f /etc/nginx/sites-enabled/default
sudo ln -s /opt/xingyue_v2/nginx.conf /etc/nginx/sites-enabled/diary.conf
sudo systemctl restart nginx
```
